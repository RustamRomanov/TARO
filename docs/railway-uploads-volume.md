# Railway Volume для загрузок ASTROV

**Актуализация:** апрель 2026.

Аватары и вложения поддержки пишутся в каталог, задаваемый переменной **`ASTROV_UPLOADS_DIR`**. Если её нет, используется `app/static/uploads` внутри контейнера (при деплое данные теряются).

## Шаги в Railway

1. Откройте сервис с приложением ASTROV (тот же, где Docker / `start.sh`).
2. Вкладка **Settings** → раздел **Volumes** (или через canvas: **Add Volume**).
3. Создайте том и укажите **Mount path**, например:  
   **`/data/astrov-uploads`**
4. В переменных окружения сервиса добавьте:  
   **`ASTROV_UPLOADS_DIR=/data/astrov-uploads`**
5. Задеплойте заново. После этого новые файлы попадают на том и сохраняются между деплоями и рестартами.

Один том на один сервис; путь монтирования и значение `ASTROV_UPLOADS_DIR` должны совпадать.

## Примечание

Уже записанные в старом слое контейнера файлы на том не появятся сами: пользователям с «битой» ссылкой на аватар достаточно загрузить фото снова.
