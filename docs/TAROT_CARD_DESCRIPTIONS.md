# Визуальные описания карт Таро (сканирование)

**Актуализация структуры проекта:** апрель 2026. Тематический документ; общая архитектура: [README.md](./README.md).

## Где лежит текстовая информация сканирования

**Файл:** `app/data/tarot_card_descriptions.json`

Структура: по колодам (`deck_id`), внутри — по картам (имя файла **без расширения**). Значение — текст на русском с визуальным описанием карты (что нарисовано: символы, фигуры, цвета), полученный через Vision API.

Пример фрагмента:

```json
{
  "oriens_animal": {
    "00-Major-Fool": "На карте изображена большая медуза с прозрачным куполом...",
    "01-Major-Magician": "..."
  },
  "dark_mansion": { ... }
}
```

**Как посмотреть несколько карт:** откройте `app/data/tarot_card_descriptions.json` в редакторе или через браузер (если есть раздача статики). Ключи внутри каждой колоды — это `card_id` (имя файла без `.jpg`/`.png`). После ручного переименования файлов ключи в JSON могут не совпадать с текущими именами (остались старые хеши). Тогда поиск описания по карте не сработает, пока не перезапустите сканирование:

```bash
cd ASTROV && . venv/bin/activate && python scripts/analyze_tarot_cards.py
```

Скрипт перезапишет описания, используя **текущие** имена файлов в папках колод.

## Как используется на бэкенде

При толковании расклада (`/api/tarot/draw-batch`) бэкенд подставляет в промпт к модели визуальное описание карты из этого файла (по `deck_id` и нормализованному `card_id`), чтобы интерпретация учитывала то, что действительно изображено на карте.

См. также: `docs/TAROT_KNOWLEDGE.md`, `docs/TAROT_VISUAL_ANALYSIS.md`.
