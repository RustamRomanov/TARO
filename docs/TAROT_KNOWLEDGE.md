# Таро: сканирование карт и база знаний (книги)

**Актуализация структуры проекта:** апрель 2026. Тематический документ; общая архитектура: [README.md](./README.md).

## 1. Сканирование карт колоды

Карты в папках колод (например `frontend/src/assets/taro/Oriens Animal Tarot/`) могут иметь произвольные имена файлов. Чтобы ИИ-таролог давал точные толкования, каждую карту нужно один раз «прогнать» через Vision API и сохранить **подробное визуальное описание** в `app/data/tarot_card_descriptions.json`. По этому описанию бэкенд потом подставляет контекст в запрос к модели (без повторного сканирования).

### Запуск

```bash
cd ASTROV
source venv/bin/activate
python scripts/analyze_tarot_cards.py
```

Нужны в `.env`: `AI_API_KEY`, `AI_BASE_URL`, `AI_VISION_MODEL`. Для vision обязательна **модель с поддержкой изображений**: например `gpt-4o` или `gpt-4o-mini` (OpenAI), либо аналог вашего провайдера. Если в логе «invalid model ID» — замените `AI_VISION_MODEL` на подходящую vision-модель. Для OpenAI можно дополнительно задать `OPENAI_GPT_API_KEY` (fallback для vision).

Скрипт:

- обходит все папки колод в `frontend/src/assets/taro/`;
- для каждой карты (кроме файлов с «Рубашка» и «Колода» в имени) вызывает Vision API;
- сохраняет в `app/data/tarot_card_descriptions.json` структуру: `{ "deck_id": { "имя_файла_без_расширения": "описание..." } }`.

Поддерживаемые колоды задаются в `FOLDER_TO_DECK_ID` в скрипте (имена папок должны совпадать с папками в `taro/`).

После добавления новой колоды: добавьте её в `tarotDecks.js` и в `FOLDER_TO_DECK_ID` в скрипте, затем запустите скрипт.

## 2. Книги по таро (база знаний для ИИ)

Чтобы «скормить» ИИ книги по таро и усилить качество раскладов, удобно выделить отдельное место под тексты и потом подключать их к промптам или RAG.

### Куда класть книги

Рекомендуемая папка: **`app/data/tarot_books/`**.

- Положите туда тексты в удобном формате: `.txt`, `.md`, при необходимости — выгруженный текст из PDF.
- Имена файлов могут быть любыми (например `waite_smith.txt`, `папюс_таро.md`).

Так всё лежит рядом с `tarot_card_descriptions.json` и логикой таро в `app/`.

### Как использовать дальше

Варианты (на выбор или в комбинации):

1. **RAG (поиск по релевантным фрагментам)**  
   При запросе на толкование/итог расклада:  
   - эмбеддинги книг из `tarot_books/` (или чанки текста);  
   - по запросу пользователя и названиям карт выбираются релевантные куски;  
   - они подставляются в системный/пользовательский промпт как контекст.  
   Нужна доработка бэкенда (эмбеддинги, хранилище, выборка).

2. **Фиксированный контекст в системном промпте**  
   При старте сервиса или по требованию загружать несколько выбранных файлов из `tarot_books/`, объединять в один текст (или суммаризовать) и добавлять в system prompt для таро-эндпоинтов. Проще всего реализовать, но объём текста ограничен размером контекста модели.

3. **Отдельный шаг «подготовка знаний»**  
   Разово или по расписанию обрабатывать файлы из `tarot_books/` (например, разбивать на чанки, считать эмбеддинги, сохранять в БД или JSON). При запросе на расклад — только выборка и подстановка в промпт.

Итого: создайте папку `app/data/tarot_books/`, скопируйте туда книги (в текстовом виде), дальше можно подключать их через RAG или через системный промпт по мере доработки бэкенда.
