# Вкладка «Таро» — устройство и техническая документация

**Актуализация структуры проекта:** апрель 2026. Сводка по приложению: [README.md](./README.md). Техдок вкладки: [TAB_TARO_TECHNICAL.md](./TAB_TARO_TECHNICAL.md).

Подробное описание того, что есть на вкладке Таро, как оно расположено и как работает под капотом.

---

## 1. Общая структура экрана

Вкладка рендерится компонентом **`Tarot`** (`frontend/src/pages/Tarot.jsx`). Маршрут: **`/tarot`** (в `App.jsx`, нижняя навигация — «Таро»).

Экран делится на **два режима**, которые переключаются через `AnimatePresence`:

- **Режим веера** — когда `spread.length === 0`: показываются колода, вопрос, пресеты и веер из 7 карт с кнопками «Вытянуть карту» / «Сделать расклад» и «Перетасовать».
- **Режим результата** — когда `spread.length > 0`: показываются вытянутые карты с толкованиями, опционально «Итог расклада» и кнопка «Написать новый расклад».

Сверху вниз блоки расположены так:

1. Фон (туман, частицы)
2. Заголовок «Таро-Зал» / «Ритуальный расклад»
3. Выбор колоды (горизонтальный скролл превью)
4. Карточка с вопросом, голосовым вводом, пресетами раскладов и быстрыми вопросами
5. Либо блок веера (кнопки + веер карт), либо блок результата (сетка карт + итог + кнопка сброса)

## 2. Что где расположено (по блокам)

### 2.1. Фон

- Два слоя с классами `tarot-fog-particles` и `tarot-fog-particles--deep` (CSS в `index.css`) — создают «мистический туман».
- 18 плавающих частиц (`fog-particle`), позиции и размеры задаются случайно при монтировании (`fogParticles`).

### 2.2. Заголовок

- Подзаголовок: «Таро-Зал» (серый текст).
- Заголовок: «Ритуальный расклад» (крупный, шрифт Cinzel).

### 2.3. Выбор колоды

- Текст: «Выберите колоду».
- Горизонтальный список кнопок (`overflow-x-auto`). Каждая кнопка — превью рубашки колоды (высота `h-28`, ширина контейнера `w-[88px]`).
- При клике по выбранной колоде переключается показ описания (название + описание под превью). При клике по другой колоде — меняется активная колода и показывается её описание.
- Данные колод приходят из `tarotDecks` (`frontend/src/data/tarotDecks.js`) или, если колод нет, из локального `fallbackDecks` (Классика, Мистика, Космос с общим `arcanaDeck`).

### 2.4. Карточка «Сформулируй вопрос»

- Текст: «Сформулируй вопрос».
- **Textarea** для вопроса (placeholder: «Например: Какой урок я сейчас прохожу?»).
- Справа внизу в textarea — кнопка микрофона (🎙 / ⏹): запуск/остановка голосового ввода (Web Speech API, `ru-RU`). При записи показывается «Слушаю: …» и возможна ошибка распознавания.
- Текст: «Выберите количество карт или расклад».
- **Пресеты раскладов** (сетка 2–3 колонки):
  - 3 — «Прошлое • Настоящее • Итог»
  - 5 — «Путь и выбор»
  - 7 — «Пирамида»
  - 10 — «Кельтский крест»
- Ряд быстрых кнопок: «Карта дня», «Тема недели», «Скрытые таланты», «Препятствия», «Совет» — подставляют готовый вопрос и при необходимости `spreadSize = 1`.

### 2.5. Блок веера (когда расклада ещё нет)

- Две кнопки в ряд по центру:
  - **«Вытянуть карту»** (если выбран 1 карта) или **«Сделать расклад»** (если 3/5/7/10) — сразу вызывает `drawCard()` (без анимации перетасовки), затем показ результата.
  - **«Перетасовать»** — запускает трёхфазную анимацию: веер → колода → тасовка → снова веер (см. ниже).
- Под кнопками — область высотой `260px` с контейнером карт по центру. Внутри **7 карт** в виде веера: каждая карта — кнопка с рубашкой, при клике карта выдвигается, запрашивается одна карта с бэкенда, затем карта переворачивается и показывается результат (переход в режим результата).

### 2.6. Блок результата (после вытягивания)

- Сетка карт (`grid`, до 3 колонок на десктопе). Для каждой карты:
  - Подпись позиции (если есть для выбранного размера расклада), например «Прошлое», «Настоящее», «Итог».
  - 3D-карта: анимация появления с переворотом (рубашка → лицо). Рубашка — через `tarot-back-wrap` и `img` или градиент; лицо — изображение карты и текст.
  - Ниже — карточка с названием карты и толкованием (пока идёт запрос — «Читаю энергию карты...»).
- Если расклад из нескольких карт — после всех карт выводится блок **«Итог расклада»** с общим толкованием (отдельный запрос к API).
- Кнопка **«Написать новый расклад»** — сброс `spread`, вопроса, размера расклада и возврат к вееру.

## 3. Данные и состояние

### 3.1. Хук `useTarot()`

- `isShuffling` — идёт ли анимация перетасовки.
- `spread` — массив вытянутых карт (пустой = показываем веер).
- `setSpread` — установка результата расклада.
- `activeDeckId` / `setActiveDeckId` — выбранная колода.
- `shuffle`, `reset` — вспомогательные функции (reset очищает spread).

### 3.2. Локальный state в `Tarot`

- `question` — текст вопроса.
- `spreadSize` — количество карт (1, 3, 5, 7, 10).
- `isDrawing` — идёт запрос к API (расклад или одна карта из веера).
- `overallInterpretation` — текст «Итог расклада» для многокарточного расклада.
- `showDeckInfo` — показывать ли описание колоды под превью.
- Голос: `isRecording`, `liveTranscript`, `speechError`, `speechSupported`, `recognitionRef`, `recordingBaseRef`.
- Анимации веера:
  - `shufflePhase`: `'fan' | 'stack' | 'spread'` — фаза перетасовки (веер / колода / раскрытие обратно).
  - `shuffleJitter` — массив смещений `{ x, y, rotate }` для карт в фазе «колода», чтобы визуально «тасовать».
  - `flippingCardIndex` — индекс карты в веере, которую пользователь выбрал кликом (выдвигается и переворачивается).
  - `drawnCardForFan` — объект одной вытянутой карты для этой выбранной позиции (имя, толкование, изображение и т.д.).

### 3.3. Колоды и карты

- **Источник колод**: `tarotDecks` из `frontend/src/data/tarotDecks.js`. Строится через `import.meta.glob` по папкам в `frontend/src/assets/taro/` (Oriens Animal Tarot, The Nightmare Before Christmas Tarot и др.). Файл с «рубаш» в имени — рубашка колоды (`backImage`), с «колод» — превью колоды (`deckImage`), остальные — карты. Имена карт заданы единообразно: `00-Major-Fool.jpg` … `21-Major-World.jpg`, затем `22-Wands1.jpg` … `35-Wands14.jpg`, `36-Cups1` … `49-Cups14`, `50-Swords1` … `63-Swords14`, `64-Pentacles1` … `77-Pentacles14.jpg` (скрипт переименования: `scripts/rename_tarot_cards.py`).
- Если реальных колод нет (`tarotDecks.length === 0`), используются `fallbackDecks` с общим списком 22 арканов (`arcanaDeck`) и градиентными рубашками.
- **Веер**: показываются первые 7 карт из `cardPool` текущей колоды (`deckView = cardPool.slice(0, 7)`). Углы поворота заданы массивом `fan`: `[-26, -16, -6, 6, 16, 26, 36]` градусов. Это только визуал веера; при вытягивании карта выбирается случайно из всей колоды (см. логику API и `drawSingleCard` / `drawCard`).

## 4. Как технически работает логика

### 4.1. Выбор колоды

- `activeDeck` = `decks.find(d => d.id === activeDeckId)`.
- `cardPool` = `activeDeck.cards` (или `arcanaDeck` для fallback).
- При смене колоды веер и рубашки перерисовываются под новую колоду (рубашка из `activeDeck.backImage` или градиент).

### 4.2. Одна карта по клику в веере

1. Пользователь нажимает на одну из 7 карт веера.
2. Проверки: не идёт `isDrawing`, `flippingCardIndex === null`, не идёт перетасовка.
3. `setFlippingCardIndex(index)` — эта карта считается «выбранной».
4. Тактовая отдача (Telegram HapticFeedback или `navigator.vibrate`).
5. Вызов **`drawSingleCard()`**:
   - из `cardPool` случайно выбирается одна карта (`shuffled[0]`);
   - отправляется **POST `/api/tarot/draw`** с `deck`, `question`, `card_name`, `position: 1`, `total: 1`;
   - в ответ приходят `card_name` и `interpretation`; формируется объект карты с `image` (из колоды или placehold.co).
6. `setDrawnCardForFan(result)` — результат привязывается к выбранной карте.
7. Анимация: карта уже выехала вперёд (`isPullingOut`), внутренний `motion.div` переворачивается с `rotateY(180)` на `0` (рубашка → лицо с пришедшей картой).
8. Через ~1300 ms: `setSpread([result])`, сброс `flippingCardIndex` и `drawnCardForFan` — экран переключается на режим результата с одной картой.

### 4.3. Кнопка «Вытянуть карту» / «Сделать расклад»

- Вызывается **`drawCard(options)`** без смены фазы перетасовки.
- `drawCard`:
  - Берёт `spreadSize` и `question` (можно переопределить через `options`).
  - Перемешивает `cardPool`, берёт первые `spreadSize` карт, кладёт в `spread` с `loading: true` и сразу показывает экран результата.
  - Последовательно для каждой карты: задержка 1.3 s (кроме первой), POST `/api/tarot/draw` с `position` и `total`, обновление элемента в `spread` (имя, толкование, изображение).
  - Для расклада из нескольких карт — ещё один POST для «Общий итог расклада», результат в `overallInterpretation`.
- Так интерпретации приходят по одной и карты «подгружают» текст без лишней нагрузки на API (снижение риска 429).

### 4.4. Кнопка «Перетасовать»

- Доступна только при `shufflePhase === 'fan'` и не во время `isShuffling`.
- Включение: `setIsShuffling(true)`, `setShufflePhase('stack')`, `setShuffleJitter([])`.
- Звук перетасовки (Web Audio API, короткий тон).
- Таймеры:
  - 550 ms: обновление `shuffleJitter` (случайные небольшие x, y, rotate для каждой карты) + звук — карты визуально «тасуются» в стопке.
  - 950 ms: новое значение `shuffleJitter` + звук — второй этап тасовки.
  - 1400 ms: `setShufflePhase('spread')`, `setShuffleJitter([])` — карты разъезжаются обратно в веер.
  - 2000 ms: `setShufflePhase('fan')`, `setIsShuffling(false)`.
- Позиции карт считаются так:
  - **fan / spread**: `x = (index - 3) * 10`, `y = 0`, `rotate = fan[index]`.
  - **stack**: `x = shuffleJitter[i].x`, `y = shuffleJitter[i].y`, `rotate = shuffleJitter[i].rotate` (или 0 до первого jitter).

### 4.5. Рубашка карты (на всю карту)

- Рубашка не рисуется через один только `background-image` на div (из-за этого раньше могла казаться «полоской»).
- Используется обёртка **`tarot-back-wrap`** (CSS в `index.css`): `position: absolute`, `inset: 0`, `min-width/height: 100%`, внутри при наличии `backImage` — тег **`<img>`** с `object-fit: cover`, `object-position: center`, чтобы изображение всегда заполняло область карты.
- Если `backImage` нет — внутри `tarot-back-wrap` рисуется div с градиентом на всю область. Так сделано и в веере, и в блоке результата расклада.

### 4.6. 3D-переворот карты

- У карты два «лица»: `.tarot-face--back` (rotateY 180deg) и `.tarot-face--front` (0deg), оба с `backface-visibility: hidden`.
- Родитель с `transform-style: preserve-3d` и `perspective` задаёт 3D-контекст. Анимация переворота — изменение `rotateY` с 180 на 0 (или наоборот) у родителя, чтобы было видно то рубашку, то лицо.

## 5. API бэкенда

- **Endpoint**: `POST /api/tarot/draw`.
- **Тело запроса** (TarotRequest):
  - `deck` — id колоды (например `hermetic`),
  - `question` — вопрос пользователя,
  - `card_name` — название карты (если уже известна; иначе бэкенд выбирает случайную из 22 арканов),
  - `position`, `total` — позиция в раскладе и общее число карт (для контекста толкования).
- **Ответ** (TarotResponse): `card_name`, `interpretation`.
- На бэкенде (`app/api/routes.py`): из payload берётся или задаётся `card_name`, формируется промпт для ИИ (таролог, карта, колода, позиция, вопрос), вызывается `ai_client.generate_text`, возвращается имя карты и толкование.

## 7. Краткая схема потоков

- **Клик по карте в веере** → выдвижение карты → `drawSingleCard()` → POST `/api/tarot/draw` (1 карта) → переворот карты → через 1.3 s переход на экран результата с одной картой.
- **«Вытянуть карту» / «Сделать расклад»** → `drawCard()` → сразу `setSpread` с N картами → последовательные POST для каждой карты и один для итога → экран результата с постепенной подгрузкой толкований.
- **«Перетасовать»** → анимация fan → stack (с jitter) → spread → fan; без вызова API и без смены `spread`.
- **«Написать новый расклад»** → `reset()`, очистка вопроса и размера → `spread = []` → снова показывается веер и форма вопроса.

Вот так устроена и работает вкладка Таро в текущей реализации.
